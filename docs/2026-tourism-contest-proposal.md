## 1. 들어가며 — 매주 반복되는 한 가지 고민

금요일 저녁이 되면 누구나 한 번쯤 떠올리는 질문이 있습니다.

> **"이번 주말, 어디로 가지?"**

검색창에 "주말 가볼 만한 곳"을 치면 블로그 수십 개가 쏟아집니다. 하나하나 열어 사진을 보고, 운영시간을 확인하고, 거리를 재고, 비 예보를 체크하고, 친구와 톡으로 의견을 주고받습니다. 그러는 사이 토요일이 되어버리고, 결국 **늘 가던 카페**로 돌아갑니다.

이 고민의 본질은 정보 부족이 아닙니다. **한국관광공사 TourAPI 만 해도 전국 수십만 건의 관광 정보가 정제된 형태로 무료 공개**되어 있습니다. 진짜 문제는 "어디 가야 하지?"라는 질문에 답하기 위해 사용자가 직접 수행해야 하는 **의사결정 노동**입니다.

이모추는 그 노동을 AI 가 대신 합니다. **위치·기분·시간·동반자·취향 5가지만 알려주시면**, 한국관광공사의 실시간 공식 데이터를 분석해 동선까지 고려한 코스를 10초 만에 설계해드립니다.

## 2. 솔루션 — 다섯 번의 탭으로 끝나는 코스 설계

### 2.1 사용자 플로우

```
홈 화면
  ↓ "코스 만들기"
1단계 [목적지]      내 주변 GPS · 도시 선택 · 분위기 추천
  ↓
2단계 [기분]        피곤·에너지·로맨틱·힐링·모험·맛집 6가지
  ↓
3단계 [시간]        반나절·종일·여유·1박2일
  ↓
4단계 [동반자]      혼자·연인·가족·친구
  ↓
5단계 [취향]        자연·맛집·문화·카페·액티비티·사진 (다중)
  ↓
[제출] → 10초 로딩 → 코스 결과 (타임라인 + 지도)
  ↓ "카카오톡 공유"
친구에게 영구 URL 전송
```

### 2.2 첫 인상 — 매거진처럼 정돈된 홈 화면

데스크톱에서 이모추를 열면 가장 먼저 보이는 것은 **에디토리얼 스타일의 hero 사진**과 **8 : 4 비율 매거진 레이아웃**입니다.

좌측 본문에는:
- 추천 관광지 그리드 (3 × 2)
- AI 코스 추천 CTA 카드
- 이번 주말 근처 축제 가로 스크롤

우측 사이드바에는:
- 토·일 날씨 카드
- 축제 사이드 리스트
- 위치 변경 패널

**모바일에서는 사이드바 콘텐츠가 본문 흐름에 자연 합류**하여 단일 컬럼으로 정돈됩니다. "데스크톱에서 좁은 모바일 폼을 보는" 답답함을 제거하고, 화면 크기에 맞는 정보 밀도를 제공합니다.

### 2.3 코스 결과 — 타임라인과 지도가 함께 움직이는 split view

코스가 완성되면 화면이 두 영역으로 나뉩니다.

- **좌측**: 시간순 타임라인. 각 장소 카드는 역할 컬러(관광지 / 맛집 / 카페 / 축제 / 숙박) 로 구분된 번호 뱃지 + 이미지 + 설명 + 다음 장소까지의 이동 시간·거리.
- **우측**: 카카오맵 sticky 뷰. 마커는 카드와 동일 컬러로, 모든 장소가 한눈에.

**카드와 마커는 양방향 연동**됩니다. 카드를 클릭하면 지도가 해당 마커로 panTo + ring 하이라이트, 마커를 클릭하면 해당 카드가 부드럽게 스크롤되어 화면 중앙으로 옵니다. **1박2일 코스는 1일차 / 2일차 탭으로 분리**되어 한 번에 한 일차의 동선만 명확하게 표시됩니다.

장소 카드를 다시 한 번 클릭하면 상세 모달이 열립니다. 이미지 갤러리·운영시간·전화·홈페이지·편의시설(주차/유모차/키즈/반려동물) 정보가 2-column 레이아웃으로 정리되며, **URL 은 `/spot/{contentId}` 로 변경**되어 그 상태에서 새로고침해도 같은 정보가 풀 페이지로 표시됩니다 (Intercepting Routes).

## 3. 기술적 차별점

### 3.1 TourAPI 4.0 KorService2 — 11개 엔드포인트 통합 활용

공모전 필수 요건은 **TourAPI 3개 이상 활용**이지만, 이모추는 **11개**를 사용합니다. 단순 호출이 아니라, 각 API 의 응답을 단일 코스 객체로 통합하는 데이터 파이프라인을 설계했습니다.

| API | 용도 |
|-----|------|
| `searchFestival2` | 주변 축제 검색 (날짜 필터링) |
| `locationBasedList2` | 위치 기반 관광지/음식점/숙박 |
| `areaBasedList2` | 지역 기반 관광지 |
| `detailCommon2` | 장소 공통 상세정보 |
| `detailIntro2` | 운영시간 · 편의시설 추출 |
| `detailImage2` | 이미지 갤러리 |
| `detailInfo2` | 반복 정보 (코스, 객실 등) |
| `searchKeyword2` | 키워드 검색 |
| `searchStay2` | 숙박 (1박2일 코스용) |
| `areaCode2` | 지역 코드 |
| `categoryCode2` | 분류 코드 |

데이터 활용 깊이를 보여주는 한 가지 예: **장소 상세 페이지의 편의시설 뱃지**(주차·유모차·키즈·반려동물·운영시간) 는 `detailIntro2` 응답을 파싱해서 가족·반려동물 동반 사용자가 즉시 확인할 수 있게 가공한 결과입니다. TourAPI 가 제공하는 비정형 텍스트(`parking`, `chkbabycarriage` 등 25+ 필드) 에서 의미 있는 5개 신호를 추출하는 경량 분류기를 lib 레이어에서 자체 구현했습니다.

### 3.2 AI 엔진 — Gemini 가 동선·시간·날씨를 동시에 고려

코스 생성은 단순한 "관광지 5개 추천"이 아닙니다. AI 엔진은 다음 순서로 동작합니다.

```
1. 후보 수집 (TourAPI 다중 호출)
   ↓ 위치 반경 50km 내 관광지 / 맛집 / 카페 / 축제 / 숙박 후보 추출
2. 사전 스코어링
   ↓ 역할별 점수 + 사용자 취향 가중치 + 감정 가중치
   ↓ 편의시설 보너스(가족=키즈+유모차, 반려=pet) + 날씨 가중치(맑음=야외, 비=실내)
3. detailIntro2 보강
   ↓ 후보 상위 N개에 대해 운영시간·편의시설 자동 추출
4. Gemini 코스 생성
   ↓ 스코어링된 후보 + 사용자 조건 + 기상청 날씨 → JSON 코스 출력
   ↓ contentId · 시간 · 동선 거리 검증
5. 검증 실패 시 폴백 코스 생성 (deterministic 알고리즘)
```

특히 **AI 응답 검증과 폴백 로직**이 핵심입니다. Gemini 가 환각(존재하지 않는 contentId 반환, 시간 순서 뒤집힘, 동선이 200km 넘는 비현실적 코스 등)을 일으키더라도 사용자에겐 항상 정상 코스가 도달합니다. 환각 방지 가드레일을 코스 생성 파이프라인에 내장한 것은 LLM 기반 서비스의 신뢰성에 직접 영향을 주는 부분입니다.

### 3.3 Next.js App Router 고급 기능 — Intercepting Routes + Parallel Routes

이모추는 Next.js 16 App Router 의 두 가지 고급 라우팅 기능을 채택했습니다.

```
app/
  spot/[contentId]/page.tsx              ← 전용 페이지 (server component, OG 메타 생성)
  @modal/
    default.tsx                          ← Parallel Route 빈 slot
    (.)spot/[contentId]/page.tsx         ← 인터셉트된 모달 (client)
```

`(.)spot` 의 `(.)` 접두는 "같은 레벨 intercept" 를 뜻합니다. `/festival` · 홈 · 코스 결과 어디서 장소 카드를 누르든:

- **Link 클릭 시**: 인터셉트되어 모달 오버레이로 표시. URL 은 `/spot/{id}` 로 바뀌지만 사용자는 부드러운 모달 UX 경험. ESC·뒤로가기로 닫힘. 스크롤 위치 보존.
- **새로고침 / 직접 접속 시**: 전용 페이지로 풀 화면 렌더 + OG 메타 태그 자동 생성. 카카오톡으로 공유받은 사람도 깔끔한 페이지 진입.

이 패턴은 인스타그램 사진 모달과 동일한 구조이며, **"공유 가능한 URL + 모달 UX"** 라는 모순을 해결합니다. 사용자는 자연스럽게 콘텐츠를 둘러보다가, 마음에 드는 장소를 발견하면 카카오톡으로 친구에게 보내고, 받은 친구는 검색엔진에 색인된 페이지로 진입할 수 있습니다.

### 3.4 디자인 시스템 — 토큰·컬러팔레트·역할 기반 컬러

- **컬러**: 따뜻한 베이지(#FAF7F2) 위에 딥 테라코타(#C5532D) 와 모카(#8B5E3C) 가 액센트. 옛 AI 서비스의 흔한 오렌지·핑크·바이올렛 그라데이션을 모두 제거하고, 에디토리얼 톤으로 통일.
- **타이포**: 헤드라인 Gmarket Sans Bold, 본문 Pretendard, 로고만 CookieRun. 한글 가독성과 브랜드 톤을 동시 확보.
- **역할 컬러 (Phase 2 신규)**: 코스의 각 장소가 관광지 / 맛집 / 카페 / 축제 / 숙박 5종 중 어느 역할인지에 따라 번호 뱃지·지도 마커·라벨이 동일 컬러로 통일. 사용자는 코스를 한눈에 봐도 "이 코스는 카페가 2개, 맛집이 1개" 같은 구성을 색으로 즉시 파악.
- **반응형**: 1024px (lg) 를 분기점으로 데스크톱은 매거진 split view, 모바일은 단일 컬럼. 두 레이아웃은 정보 손실 없이 자연 전환.

### 3.5 접근성

- 모든 카드는 `<button>` 또는 `<Link>` 로 키보드 접근 가능, `aria-label` · `aria-pressed` · `aria-current="page"` 적절 부여
- SpotDetail 모달은 **focus trap** 구현 (오픈 시 X 버튼으로 자동 포커스, Tab/Shift+Tab 으로 모달 내부 순환, 닫을 때 원래 트리거 요소로 포커스 복원, `role="dialog"` + `aria-modal="true"` + `aria-label`)
- DayTabs · FestivalFilterBar 는 `role="tablist"` + `role="tab"` + `aria-selected` 로 ARIA 표준 준수
- 스크린리더용 `sr-only` 라벨이 아이콘만 표시되는 편의시설 뱃지에 부착되어 시각장애 사용자도 정보 동등 접근

## 4. 데이터 흐름과 아키텍처

```
[사용자 브라우저]
  │
  │ 1. 위치 + Wizard 5단계 응답 입력
  ▼
[Next.js Client (React 19)]
  │
  │ 2. POST /api/course (lat, lng, duration, companion, preferences,
  │                       feeling, destinationType, cityAreaCode, mood)
  ▼
[Next.js API Route (server)]
  │
  ├─ 3a. 기상청 단기예보 API → 날씨 추출
  │
  ├─ 3b. TourAPI 4.0 다중 호출
  │       ├ searchFestival2 (반경 N km)
  │       ├ locationBasedList2 (관광지·음식점·숙박)
  │       ├ areaBasedList2 (지역 기반)
  │       └ detailIntro2 (편의시설 보강)
  │
  ├─ 4. 사전 스코어링 (lib/weekend-ai.ts)
  │       역할 + 취향 + 감정 + 편의시설 + 날씨 가중치 점수
  │
  ├─ 5. Google Gemini 2.5 Flash → JSON 코스 생성
  │       (스코어링 후보 + 조건 + 날씨 컨텍스트 전달)
  │
  ├─ 6. AI 응답 검증
  │       contentId 유효성 + 시간순서 + 동선 거리
  │       실패 시 deterministic 폴백 코스 생성
  │
  ├─ 7. Supabase saved_courses 테이블에 영구 저장
  │       courseId · shareUrl · course_data JSON
  │
  ▼
[사용자에게 응답]
  CourseResponse { courseId, shareUrl, course: { title, summary, stops[]... } }
  │
  ▼
[/course/{slug}] split view 결과 페이지
  ├ 좌: Timeline (StopCard × N)
  ├ 우: 카카오맵 sticky (역할 컬러 마커)
  └ 카카오톡 공유 / 링크 복사
```

**아키텍처 특징**
- API 키는 모두 서버사이드에서만 사용 (`NEXT_PUBLIC_` 접두 없는 환경변수)
- ISR (`revalidate: 3600`) 로 동일 장소 반복 조회 시 TourAPI 호출 최소화
- next/image + Vercel CDN 으로 LCP 최적화 (TourAPI 의 http URL 도 unoptimized fallback 으로 호환)
- Supabase 영구 저장으로 공유 받은 사람이 1년 뒤에도 같은 코스 조회 가능

## 5. UI/UX 하이라이트

### 5.1 타임라인 프리뷰 스켈레톤 로딩

코스 생성에는 평균 15~25초가 소요됩니다. 이 시간 동안 사용자에게 무엇을 보여줄지가 이탈률을 좌우합니다. 이모추는 **결과 페이지의 split view 레이아웃을 미리 깔고, 스톱 카드 자리에 스켈레톤이 800ms 간격으로 순차 등장**하는 방식을 채택했습니다.

상단에는 4개의 안내 메시지가 8초 간격으로 순환합니다:
1. "주변 관광지를 살펴보고 있어요"
2. "AI가 코스 순서를 계산하는 중이에요"
3. "실시간 날씨와 동선을 반영하고 있어요"
4. "마지막으로 다듬는 중이에요"

응답이 도착하면 같은 자리에 실제 카드가 crossfade 로 교체되어, **로딩 → 결과 전환이 페이지 이동이 아닌 데이터 채워짐**으로 느껴집니다. 사용자는 "기다린다"가 아니라 "지켜본다"는 경험을 합니다.

### 5.2 5단계 위저드 — 데스크톱 2-column / 모바일 풀스크린 전환

데스크톱에서는 좌측에 진행 스테퍼(완료 단계 클릭으로 즉시 수정 가능), 우측에 현재 단계 질문. 모바일에서는 상단 슬림 프로그레스 바 + 전체 화면 한 단계씩 전환 + 하단 sticky Nav. 두 환경의 UX 패러다임을 분기하면서도 동일한 단일 reducer 로 상태 관리.

### 5.3 위치 변경 모달

GPS 거부 사용자, 다른 도시 여행 계획자를 위한 위치 변경 모달. 검색·GPS 재요청·자주 쓴 위치(localStorage) 3가지 진입 경로를 단일 모달에 통합. 데스크톱은 가운데 모달, 모바일은 바텀 시트 (키보드 가림 방지 스크롤 보정 포함).

### 5.4 축제 D-Day 뱃지 시스템

축제는 시간 민감 정보입니다. `/festival` 매거진 그리드 카드의 좌상단에는 다음 4가지 상태 중 하나가 뱃지로 표시됩니다:

- **"진행 중"** (success-soft 톤): 오늘이 축제 기간에 포함
- **"마감 임박"** (warning-soft 톤): 종료일 D-3 이내
- **"이번 주말"** (brand-soft 톤): 이번 토·일에 진행
- **"D-N"** (brand-soft 톤): 미래 축제 카운트다운

사용자는 "지금 가야 하는 축제"와 "여유 있게 계획할 축제"를 한 번의 시선으로 분류할 수 있습니다.

## 6. 발전 계획

이모추는 공모전 출품 시점이 끝이 아니라 출발점입니다. 다음 4단계의 로드맵을 이미 설계해두었습니다.

### Phase 5: 글로벌 관광객 확장 (다국어 영어·일본어)
한국관광공사 TourAPI 의 공식 다국어 서비스(EngService / JpnService)를 연동하여 외국인 관광객 대상 영어·일본어 버전 출시. UI 문자열 100+건은 `next-intl` dictionary 로 번역, 경로는 `/en/...` · `/ja/...` 로 분기. `hreflang` 메타 태그로 SEO 대응. **국가별 검색엔진에서 한국 관광 정보를 발견할 새 진입점**이 됩니다.

### Phase 6: 지도 중심 UX
`/festival` 페이지에 좌 목록 / 우 sticky 카카오맵 split view 도입. SpotDetail 내 지도 미니뷰 임베드로 외부 앱 전환 없이 위치 파악. 이미지 full-screen viewer (탭하여 확대) 로 관광지 사진의 디테일 감상.

### Phase 7: 개인화 / 소셜
저장한 코스 마이페이지 (BottomTabBar 4번째 탭), 코스 방문 체크·사진 업로드·리뷰 기능. 익명 브라우저 ID 기반으로 회원가입 부담 없이 즉시 사용. 코스 수정 UX (드래그 재정렬 / AI 재생성 요청).

### Phase 8: 기술 고도화
Gemini Streaming(SSE) 으로 코스 점진 생성 — 사용자가 첫 장소부터 즉시 보기 시작. PDF 다운로드 / OG 동적 이미지 생성. TourAPI 클라이언트 LRU 캐시. Edge Runtime 전환. Playwright 시각 회귀 + Lighthouse CI + WCAG AA 접근성 감사 자동화.

상세 로드맵은 저장소의 `docs/superpowers/2026-04-25-post-launch-backlog.md` 에 통합 정리되어 있습니다.
