## 1) 서비스 기획배경 및 필요성

### 1. 서비스 기획 배경

대한민국 사람이라면 누구나 매주 한 번씩 똑같은 질문을 떠올립니다. **"이번 주말, 어디로 가지?"** 검색·블로그·인스타그램을 헤매다 결국 평소 가던 곳으로 돌아가는 일이 흔합니다. 이 고민의 본질은 **정보 부족**이 아닙니다. 한국관광공사 TourAPI 만 해도 전국 수십만 건의 정제된 관광 정보가 무료로 공개되어 있습니다.

진짜 문제는 **의사결정 노동**입니다. 가볼 만한 곳·맛집·축제·날씨·이동 시간·운영시간·편의시설을 사용자가 직접 종합해야 합니다. 이 노동은 매주 반복되며, 단 1~2시간 안에 결론을 내지 못하면 결국 "익숙한 카페"로 돌아갑니다.

이모추는 이 의사결정을 **단 5번의 탭**으로 압축합니다. 위치·기분·시간·동반자·취향만 알려주시면 한국관광공사의 실시간 공식 데이터를 분석해 동선까지 고려한 코스를 10초 만에 설계합니다.

### 2. 서비스 필요성

| 시장의 빈 자리 | 이모추가 채우는 가치 |
|---|---|
| 검색·블로그 정보 종합 노동 | AI 가 11개 TourAPI 엔드포인트를 자동 종합 |
| 단편적 추천 (장소 1개씩) | 시간 흐름이 담긴 **코스** (오전·점심·오후·숙박) |
| 정적 정보 (운영시간·날씨 미반영) | 기상청 단기예보 + 운영시간 + 편의시설 동시 고려 |
| 공유 어려움 (카톡으로 링크 잘림) | 코스마다 영구 URL + OG 메타 태그 자동 생성 |
| 외국인 관광객 진입 장벽 | (Phase 5) TourAPI EngService/JpnService 다국어 확장 예정 |

공모전 주제 *"데이터 가치를 넘어, 서비스로 실현하다: 관광데이터 스케일 UP"* 에 정확히 부합합니다. 이모추는 한국관광공사 데이터의 가치를 **개인 사용자 일상의 의사결정**으로 직접 연결하는 다리입니다.

## 2) 서비스 개요

### 1. 기획 서비스 소개 (한 줄)

**위치·기분·동반자에 맞는 주말 나들이 코스를 한국관광공사 TourAPI 와 Google Gemini AI 가 10초 만에 설계해 카카오톡으로 친구에게 공유까지 한 번에 끝내는 큐레이션 서비스.**

### 2. 기획 서비스 주요 기능

1. **5단계 AI 코스 위저드** — 목적지(GPS·도시·분위기) → 기분(6종) → 시간(반나절~1박2일) → 동반자(혼자·연인·가족·친구) → 취향(자연·맛집·문화·카페·액티비티·사진 다중)
2. **타임라인 + 지도 결과 화면** — 시간순 코스 카드(역할 컬러 5종으로 관광지/맛집/카페/축제/숙박 구분) + sticky 카카오맵, 카드 ↔ 마커 양방향 연동, 1박2일은 일차별 탭 분리
3. **장소 상세 페이지** — 이미지 갤러리·운영시간·전화·홈페이지·편의시설(주차/유모차/키즈/반려동물) 통합 제공. URL 공유 가능 + 부드러운 모달 UX 동시 지원 (Next.js Intercepting Routes)
4. **카카오톡 공유** — 코스마다 영구 URL 발급, OG 썸네일·제목·설명 자동 생성으로 친구·가족과 자연스러운 협업
5. **축제 매거진 페이지** — 진행 중·이번 주말·곧 시작·D-Day 4단계 상태 뱃지로 시간 민감 정보 즉시 파악
6. **반응형 매거진 디자인** — 데스크톱 8/4 split view (좌 본문·우 사이드바), 모바일 단일 컬럼 자연 합류

### 3. 서비스 차별성

**(가) TourAPI 11개 엔드포인트 통합 활용** — 공모전 필수 요건(3개 이상)의 **3.7배**. 단순 호출이 아니라 각 API 응답을 단일 코스 객체로 통합하는 데이터 파이프라인을 자체 설계.

**(나) AI 환각 방지 검증·폴백 파이프라인** — Gemini 가 존재하지 않는 contentId 반환·시간 역전·비현실적 동선(>200km) 등 환각을 일으켜도 사용자에겐 항상 정상 코스 도달. LLM 기반 서비스의 신뢰성을 가드레일로 보장.

**(다) Next.js 16 App Router 고급 기능 채택** — Intercepting Routes + Parallel Routes 패턴으로 "공유 가능한 URL + 부드러운 모달 UX" 라는 모순을 해결. 인스타그램 사진 모달과 동일한 패턴.

**(라) 에디토리얼 디자인 시스템** — 토큰화된 컬러 팔레트(베이지+딥테라코타+모카) · Gmarket Sans+Pretendard 타이포 · 5가지 역할 기반 컬러. AI 서비스의 흔한 오렌지·핑크 그라데이션을 거부한 매거진 미감.

**(마) 접근성** — WCAG AA 수준의 focus trap·키보드 네비·ARIA 시맨틱·sr-only 레이블·`aria-current="page"` 등을 시각장애·키보드 사용자까지 고려.

### 4. (지역 특화 서비스의 경우 작성 필수) 서비스 내 지역 특화 관련 사항

**해당 없음.** 이모추는 전국 단위 서비스이며, 한국관광공사 TourAPI 의 17개 시·도 데이터를 모두 활용합니다. GPS 또는 도시 선택을 통해 어느 지역이든 추천이 가능하며, 특정 지자체와의 협력 또는 단일 지역 한정 자원 활용은 없습니다.

## 3) 데이터 활용 방안 (한국관광공사 OpenAPI 활용 필수)

### 1. 활용 예정 한국관광공사 OpenAPI

**TourAPI 4.0 KorService2 — 11개 엔드포인트 통합 활용** (2026-04-25 운영 기준)

| API 엔드포인트 | 활용 용도 | 호출 주기 |
|---|---|---|
| `searchFestival2` | 주변 축제 검색 (날짜·반경 필터) | 코스 생성 시·축제 페이지 |
| `locationBasedList2` | 위치 기반 관광지·음식점·숙박 후보 | 코스 생성 시 |
| `areaBasedList2` | 지역 기반 관광지 후보 | 코스 생성 시 |
| `detailCommon2` | 장소 공통 상세 (제목·주소·소개·좌표) | 장소 상세 페이지 |
| `detailIntro2` | 운영시간·주차·유모차·반려·키즈 추출 | 장소 상세 + 코스 보강 |
| `detailImage2` | 이미지 갤러리 | 장소 상세 |
| `detailInfo2` | 코스·객실 등 반복 정보 | 장소 상세 |
| `searchKeyword2` | 검색바 키워드 검색 | 검색 시 |
| `searchStay2` | 1박2일 코스용 숙박 검색 | 코스 생성 (overnight) |
| `areaCode2` | 지역 코드 매핑 | 위치 처리 |
| `categoryCode2` | 분류 코드 매핑 | 데이터 정제 |

**보조 공공/민간 API**: 기상청 단기예보 API · Kakao Maps SDK · Google Gemini API · Supabase.

### 2. 데이터 활용 방식

```
[사용자 5단계 입력]
       ↓
[POST /api/course]
       ↓
[① 기상청 단기예보 API → 토·일 날씨 추출]
       ↓
[② TourAPI 다중 호출 — searchFestival2 + locationBasedList2 +
    areaBasedList2 + searchStay2 (1박2일 시)]
       ↓
[③ 사전 스코어링 — 역할별 점수 + 사용자 취향·감정·편의시설 가중치
    + 날씨 가중치 (맑음=야외 보너스 / 비=실내 보너스)]
       ↓
[④ detailIntro2 보강 — 후보 상위 N개의 운영시간·편의시설 자동 추출]
       ↓
[⑤ Gemini 2.5 Flash → JSON 코스 생성 (스코어링 후보 + 조건 + 날씨 컨텍스트)]
       ↓
[⑥ AI 응답 검증 — contentId 유효성 + 시간순서 + 동선 거리 (≤50km)
    실패 시 deterministic 폴백 코스 생성]
       ↓
[⑦ Supabase saved_courses 영구 저장 (courseId · shareUrl · course_data JSON)]
       ↓
[사용자 결과 화면 + 카카오톡 공유 가능 URL]
```

**데이터 활용 깊이의 예시**: TourAPI `detailIntro2` 응답에는 `parking`·`chkbabycarriage`·`chkpet` 등 25+ 비정형 텍스트 필드가 있습니다. 이모추는 이를 파싱해 **5가지 의미 신호**(주차·유모차·키즈·반려동물·운영시간)로 정제하는 경량 분류기를 자체 구현했습니다. 가족·반려동물 동반 사용자가 카드 한 장에서 즉시 의사결정할 수 있도록 가공한 결과입니다.

**ISR 캐싱**: 장소 상세 페이지(`/spot/[contentId]`)는 `revalidate: 3600` 으로 1시간 단위 ISR 적용. 동일 장소 반복 조회 시 TourAPI 호출을 최소화하면서도 최신성 유지.

## 4) 서비스 발전 방향

### 1. 개발 서비스 향후 발전방향

이모추는 공모전 출품 시점이 종료가 아니라 **출발점**입니다. 5월~9월 교육·컨설팅 지원 기간을 활용해 다음 4단계 로드맵을 진행합니다.

**Phase 5 — 글로벌 관광객 확장 (다국어 영·일)** *(최우선)*
한국관광공사 TourAPI 의 **공식 다국어 서비스(EngService / JpnService)** 를 연동하여 외국인 관광객 대상 영어·일본어 버전 출시. UI 문자열 100+ 건은 `next-intl` dictionary 로 번역, 경로는 `/en/...` · `/ja/...` 분기 + `hreflang` 메타 태그 + 언어 셀렉터 UI. **국가별 검색엔진에서 한국 관광 정보를 발견할 새로운 진입점**이 됩니다. 한국 인바운드 관광 활성화에 직접 기여.

**Phase 6 — 지도 중심 UX**
`/festival` 페이지에 좌 목록 / 우 sticky 카카오맵 split view 도입, 장소 상세에 지도 미니뷰 임베드, 이미지 full-screen viewer.

**Phase 7 — 개인화 / 소셜**
저장한 코스 마이페이지(BottomTabBar 4번째 탭), 코스 방문 체크·사진 업로드·리뷰, 드래그 재정렬 기반 코스 수정 UX. 익명 브라우저 ID 기반으로 회원가입 부담 없이 즉시 사용 가능.

**Phase 8 — 기술 고도화**
Gemini Streaming(SSE) 으로 코스 점진 생성 (사용자가 첫 장소부터 즉시 보기), PDF 다운로드 / OG 동적 이미지, TourAPI 클라이언트 LRU 캐시, Edge Runtime 전환, Playwright 시각 회귀 + Lighthouse CI + WCAG AA 자동 감사.
